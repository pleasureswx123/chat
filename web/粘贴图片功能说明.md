# 粘贴图片功能说明

## 功能概述

已为 Coze 聊天界面添加了粘贴图片功能,用户可以直接使用 `Ctrl+V` (Windows) 或 `Cmd+V` (Mac) 将剪贴板中的图片粘贴到聊天框中。

## 实现原理

### 1. 目标元素定位

代码会查找 Coze SDK 渲染的 textarea 输入框:
```javascript
const textarea = document.querySelector('textarea[data-testid="bot.ide.chat_area.chat_input.textarea"]');
```

这个选择器对应的是 Coze 聊天界面的输入框:
```html
<textarea 
  data-testid="bot.ide.chat_area.chat_input.textarea" 
  placeholder="发送消息" 
  class="rc-textarea ..."
></textarea>
```

### 2. 文件上传处理

当检测到粘贴图片时:
1. 查找隐藏的 `input[type="file"]` 元素
2. 使用 `DataTransfer` API 将图片文件赋值给 input
3. 触发 `change` 和 `input` 事件,让 Coze SDK 识别文件上传

### 3. DOM 监听

使用 `MutationObserver` 监听 DOM 变化,确保在 textarea 加载后立即提示用户功能已就绪。

## 使用方法

### 方式一:截图粘贴
1. 使用截图工具(如 Windows 截图工具、Mac 截图)截取图片
2. 图片会自动保存到剪贴板
3. 在聊天界面按 `Ctrl+V` (Windows) 或 `Cmd+V` (Mac)
4. 图片会自动上传到聊天框

### 方式二:复制图片文件
1. 在文件管理器中右键点击图片文件
2. 选择"复制"
3. 在聊天界面按 `Ctrl+V` (Windows) 或 `Cmd+V` (Mac)
4. 图片会自动上传到聊天框

### 方式三:从网页复制图片
1. 在网页上右键点击图片
2. 选择"复制图片"
3. 在聊天界面按 `Ctrl+V` (Windows) 或 `Cmd+V` (Mac)
4. 图片会自动上传到聊天框

## 控制台日志

功能运行时会在浏览器控制台输出以下日志:

- `✅ 粘贴图片功能已启用` - 功能初始化成功
- `✅ Coze 聊天输入框已加载,可以使用 Ctrl+V 粘贴图片` - 输入框已就绪
- `📋 检测到粘贴图片: xxx.png image/png 123.45KB` - 检测到粘贴操作
- `✅ 图片已成功粘贴到聊天框` - 图片上传成功
- `⚠️ 未找到 Coze 聊天输入框` - 输入框未加载(可能需要等待)
- `⚠️ 未找到文件上传输入框` - 文件上传功能未启用

## 技术细节

### 事件捕获
```javascript
document.addEventListener('paste', handler, true);
```
使用捕获阶段(`true`)确保事件优先被处理。

### 阻止默认行为
```javascript
event.preventDefault();
event.stopPropagation();
```
阻止浏览器默认的粘贴行为,避免冲突。

### DataTransfer API
```javascript
const dataTransfer = new DataTransfer();
dataTransfer.items.add(file);
fileInput.files = dataTransfer.files;
```
模拟用户手动选择文件的行为。

### MutationObserver
```javascript
const observer = new MutationObserver(() => {
  const textarea = document.querySelector('...');
  if (textarea) {
    console.log('✅ 输入框已加载');
    observer.disconnect();
  }
});
observer.observe(document.body, { childList: true, subtree: true });
```
监听 DOM 变化,确保在 Coze SDK 完全渲染后提示用户。

## 故障排查

### 问题 1: 粘贴后没有反应

**可能原因**:
- Coze SDK 未完全加载
- 文件上传功能未启用

**解决方案**:
1. 等待几秒后重试
2. 检查控制台日志
3. 确认 Coze SDK 配置中 `uploadable: true`

### 问题 2: 控制台显示"未找到输入框"

**可能原因**:
- Coze SDK 版本不同,选择器可能变化

**解决方案**:
1. 打开浏览器开发者工具
2. 检查实际的 textarea 元素属性
3. 更新选择器

### 问题 3: 图片格式不支持

**支持的格式**:
- PNG
- JPEG/JPG
- GIF
- WebP

**解决方案**:
- 转换图片格式后重试
